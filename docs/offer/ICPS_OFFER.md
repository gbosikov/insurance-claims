## Что входит в подписку — полный список возможностей

### Приём и обработка документов

- **API-интеграция**: приём заявок через REST API (JSON: policy_number + ссылки на документы)
- **Автозагрузка документов**: скачивание файлов по pre-signed URL от вашей медицинской системы
- **OCR на трёх языках**: распознавание русского, грузинского и английского текста (Google Vision API)
- **AI-извлечение данных**: диагнозы (МКБ-10), суммы, даты, клиника, ФИО пациента, вид услуги
- **Кросс-документная проверка**: автоматическое обнаружение расхождений между документами одного кейса
- **Детерминированное извлечение**: опциональный режим без LLM для стандартных документов (экономия токенов)

### Анализ соответствия договору

- **RAG-поиск по договору**: поиск релевантных условий полиса по тексту диагноза и процедуры
- **Проверка исключений**: детерминированная проверка по базе exclusion rules из вординга договора
- **CARVEOUT-логика**: автоматическое применение исключений из исключений (например: диагноз исключён, но экстренный случай — покрыт)
- **Периоды ожидания**: проверка StartDate полиса и маркеров освобождения от периода ожидания
- **Суб-лимиты**: проверка остатков по каждому риску с учётом иерархии рисков
- **Положительный список процедур**: сверка оказанных услуг с явно покрытыми процедурами из договора

### Интеллектуальное принятие решений

- **Три сигнала маршрутизации**: итоговое решение = min(уверенность покрытия, качество данных, размер заявки)
- **Медицинская согласованность**: проверка соответствия диагноза, процедуры и специализации клиники
- **Extended Thinking**: углублённый анализ сложных и пограничных кейсов — активируется при сумме > 300 GEL или низкой уверенности в данных; полная цепочка рассуждений сохраняется в аудит-логе *(Anthropic Claude)*
- **Адаптивный маршрут**: AUTO_APPROVED / MANUAL_REVIEW / FRAUD_FLAG — в зависимости от уверенности

### Антифрод-проверки

- **Дублирование**: блокировка повторной подачи кейса с теми же документами
- **Частота заявок**: выявление аномально высокого числа обращений за период
- **Аномалии сумм**: обнаружение статистических выбросов по диагнозу (активируется после накопления данных)
- **Стохастический QA**: 5% автоматически одобренных кейсов отправляется на ручную проверку

### Интеграция с кор-системой Lite GROUP

- **ClaimParsing_UNI**: автоматическое создание убытка в кор-системе с прикреплёнными документами
- **Структурированный комментарий**: формируется на грузинском языке для операторов кор-системы
  `სანდოობა: 85% | დიაგნოზები: E55.9 ✓ | თანხა: 128 GEL | ...`
- **Идемпотентность**: защита от дублирующихся записей убытка при технических ретраях
- **Автоматическая аутентификация**: LiteMed API и Claims API — токены обновляются автоматически

### Операторский веб-портал

- **Список заявок**: история всех кейсов с фильтрацией по статусу, датам, полису
- **Очередь ручной проверки**: все кейсы с низкой уверенностью в одном месте
- **Детализация кейса**: пошаговая история обработки, уровни уверенности, причины решения
- **Управление апелляциями**: приём и обработка апелляций от застрахованных
- **Трекинг затрат**: стоимость OCR + AI по каждому кейсу с детализацией по шагам
- **Роли доступа**: viewer (просмотр) / operator (работа с очередью) / admin (полный доступ)

### Надёжность и безопасность

- **Dead Letter Queue**: упавшие задачи не теряются — очередь для ручного перезапуска или закрытия
- **Полный аудит-лог**: каждый шаг обработки записывается с временными метками и уверенностью
- **Маскирование ПД**: ФИО, личные номера и pre-signed URL не попадают в логи
- **API-аутентификация**: X-API-Key с rate limiting (60 запросов/мин по умолчанию)
- **JWT-аутентификация портала**: токены с временем жизни 8ч, обновление сессии
- **HMAC-подпись webhook**: верификация входящих событий от кор-системы

### Аналитика и обучение

- **Статистика точности**: процент AUTO_APPROVED / MANUAL_REVIEW, точность по типам диагнозов
- **Калибровка уверенности**: автоматическая корректировка порогов на основе результатов ручной проверки
- **Отчёт по затратам**: токены, OCR, инфраструктура — в разбивке по дням и типам документов
- **Петля обучения**: результаты ручной проверки улучшают точность будущих решений

## Что входит в установку ($3 000)

- Развёртывание системы в инфраструктуре клиента (Docker / Kubernetes)
- Интеграция с кор-системой Lite GROUP (LiteMed API + ClaimParsing_UNI)
- Загрузка справочников: МКБ-10 (12 435 записей), справочник клиник из Lite GROUP
- Индексация генеральных договоров (до 10 полисов в RAG-базу)
- Настройка пользователей и ролей в операторском портале
- Тестовый прогон на реальных кейсах (20 кейсов гарантированно)
- Обучение операторов работе с порталом (до 3 часов)

- Мониторинг и поддержание работоспособности системы
- Обновление AI-моделей и промптов при изменении условий договоров
- Добавление новых полисов в RAG-индекс (до 5 в месяц)
- Обновление справочников (МКБ-10, клиники) по запросу
- Техническая поддержка (SLA: рабочий день)
- Ежемесячный отчёт: точность, объём, динамика затрат
