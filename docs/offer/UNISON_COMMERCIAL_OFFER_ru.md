# 1. О продукте

ICPS — система автоматизированной обработки заявок ДМС: принимает документы заявки (форма 100, удостоверение личности, чек), распознаёт их, проверяет соответствие условиям договора и доступным рискам полиса и **создаёт убыток в кор-системе Lite GROUP с полным обоснованием решения** — за минуты вместо часов или дней.

```
Внешняя медсистема          ICPS                            Lite GROUP
                    ┌──────────────────────────┐
 JSON + документы → │ контроль качества → OCR  │ → getpolicylist (риски, лимиты)
                    │ → извлечение данных      │
                    │ → AI-решение по договору │ → ClaimParsing_UNI (убыток +
                    │ → антифрод               │    AI-вердикт в Comment)
                    └──────────────────────────┘
                         целевое время: ≤ 15 минут
```

Ключевой принцип: **AI никогда не отказывает в выплате самостоятельно.** Однозначные случаи одобряются автоматически, любое сомнение направляется оператору с готовым проектом решения и цитатой из договора. Каждое решение фиксируется в неизменяемом аудит-журнале (хранение 7 лет — требование регулятора).

# 2. Что входит

## Приём и контроль качества документов
- Приём заявок по API (номер медкарточки + ссылки на документы)
- Автоматическая проверка качества изображений: разрешение, размытость, освещение
- При плохом качестве — клиент сразу получает понятную причину («изображение размытое, переснимите») вместо отказа через несколько дней

## Распознавание документов (OCR)
- Google Vision + Document AI: формы, таблицы, рукописный текст
- Три языка одновременно: грузинский, русский, английский
- Пер-блочная оценка достоверности распознавания

## Извлечение данных и кросс-проверка
- AI-извлечение: ФИО, личный номер, даты, диагнозы МКБ-10, услуги, суммы
- Автоматическая сверка между документами: ФИО в форме 100 против удостоверения, диагнозы против чека, даты, учреждение — расхождения помечаются для оператора

## AI-решение по договору
- Интерпретация условий договора по **категориям** заболеваний (как написано в полисах), а не по точному совпадению кодов — с использованием полного справочника МКБ-10 (12 400+ диагнозов с иерархией)
- **Детерминированный движок исключений по вордингу**: исключения из текста страховых условий применяются до вызова AI — без риска галлюцинаций; поддерживаются диапазоны кодов МКБ-10 (F00–F99, N18.0–N18.9), CARVEOUT-оговорки («исключено, кроме ургентных случаев»), различные правила для сотрудников и членов семьи
- Поддержка условных исключений (CARVEOUT), явно покрытых процедур (POSITIVE LIST), суб-лимитов и периодов ожидания
- **Каждое решение содержит прямую цитату из договора** — оператор и аудитор видят основание
- Для сложных случаев — расширенный режим рассуждения AI с сохранением хода анализа

## Антифрод
- Дубликаты заявок (тот же пациент / дата / учреждение)
- Аномальная частота обращений
- Медицинская согласованность: соответствие услуг диагнозу (например, МРТ позвоночника при диагнозе ОРВИ — флаг оператору)

## Интеграция с Lite GROUP — готова и верифицирована
- Получение полисов, рисков, лимитов и остатков (getpolicylist) — формат выверен на реальных данных, включая особенности API
- Создание убытка (ClaimParsing_UNI) с прикреплением документов и полным AI-вердиктом в поле Comment
- Фильтрация: обрабатываются только полисы медицинского страхования
- **Это уникальное преимущество системы: повторная разработка такой интеграции заняла бы месяцы**

## Петля обучения — система становится точнее в эксплуатации
- 5% автоматически одобренных заявок выборочно перепроверяются оператором
- Система ежедневно сравнивает свою уверенность с фактической точностью и калибрует пороги
- Каждое исправление оператора фиксируется с типом ошибки — данные для целевого улучшения

## Аудит и комплаенс
- Неизменяемый журнал каждого шага обработки и каждого решения: входные данные, версия модели и промпта, использованные пункты договора, уверенность, время
- Хранение 7 лет; персональные данные маскируются в технических логах

## Рабочее место оператора
- Очередь ручной проверки с приоритетами (API; готово к подключению веб-интерфейса)
- Фиксация решения оператора в два клика: что исправлено и почему

## Безопасность
- Доступ по API-ключам с разграничением прав (внешняя система / операторы / аналитика)
- Ограничение частоты запросов, изоляция данных, белые списки доменов для скачивания документов

## Этап 1 — Теневой пилот (8 недель)

Система обрабатывает **копию** реального потока заявок параллельно с операторами. Её решения никуда не отправляются — только сравниваются с решениями людей. **Риск для бизнес-процессов Unison — нулевой.**

| Делаем мы | Нужно от Unison |
|---|---|
| Развёртывание в контуре Unison | Копия входящего потока заявок |
| Еженедельные сводки метрик | 1 оператор ~20% времени — разметка расхождений |
| Тюнинг под формулировки договоров Unison | |

## Этап 2 — Отчёт и решение (1 неделя)

Отчёт с фактическими показателями против согласованных KPI. Решение о покупке Unison принимает **по собственным цифрам**, а не по обещаниям поставщика.

## Этап 3 — Боевой запуск (2 недели, при покупке)

- Автообработка заявок до согласованного порога суммы (например, ≤200 GEL), остальное — оператору с готовым AI-вердиктом
- Поэтапное расширение порога по мере подтверждения точности
- Обучение операторов, передача документации

# 4. KPI пилота

Критерии успеха согласовываются **письменно до старта** (включая методику подсчёта и порядок разбора спорных случаев):

| Метрика | Что показывает |
|---|---|
| % совпадения решения AI с решением оператора | Точность системы |
| % заявок — кандидатов на автоодобрение | Потенциал автоматизации |
| % документов, успешно проходящих распознавание | Применимость к реальному потоку |
| p90 времени обработки | Скорость |

# 5. Технологии

- **Backend:** Python, FastAPI, Celery
- **AI:** Claude (Anthropic) — извлечение и решения; Google Vision / Document AI — распознавание
- **База данных:** PostgreSQL + pgvector (семантический поиск по договорам)
- **Очереди и кэш:** Redis
- **Развёртывание:** Docker, в контуре Unison (Linux VPS или Kubernetes); данные не покидают инфраструктуру компании, кроме обращений к AI-API
- **Миграции схемы, тесты (330+), аудит** — промышленный цикл сопровождения

- Хостинг / VPS / домены — инфраструктура предоставляется Unison

- Интеграции с кор-системами, отличными от Lite GROUP
