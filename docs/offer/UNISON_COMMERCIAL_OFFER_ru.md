# 1. О продукте

ICPS — middleware-система между медицинской системой и кор-системой страховщика. Принимает заявку (номер полиса + документы), проверяет её по условиям договора и **автоматически создаёт убыток в Lite GROUP** с полным обоснованием решения — за минуты вместо часов.

```
Медсистема               ICPS                          Lite GROUP
               ┌───────────────────────────┐
 номер полиса  │ качество → OCR → анализ  │ → getpolicylist
 + документы → │ → AI-решение по договору │ → ClaimParsing_UNI
               │ → антифрод → аудит       │   (убыток + AI-вердикт)
               └───────────────────────────┘
                    целевое время: ≤ 15 минут
```

**Ключевой принцип:** система никогда не отказывает в выплате самостоятельно. Однозначные случаи одобряются автоматически, любое сомнение направляется оператору с готовым проектом решения и прямой цитатой из договора. Каждое решение фиксируется в неизменяемом аудит-журнале (хранение 7 лет).

# 2. Пакеты

## Пилот — $8 000

Теневой режим: система обрабатывает копию реального потока заявок параллельно с операторами. Её решения не отправляются в Lite GROUP — только сравниваются с решениями людей. **Риск для бизнес-процессов — нулевой.**

**Что входит:**

| Позиция | Описание |
|---|---|
| Развёртывание в контуре клиента | Установка на предоставленный сервер (Docker), подключение к тестовому контуру Lite GROUP |
| Подготовка под реальные документы | Настройка OCR и AI-извлечения на реальных формах 100, чеках и удостоверениях Unison |
| Индексация договоров ДМС | Загрузка PDF договоров в RAG-систему, настройка интерпретации условий под формулировки Unison |
| Тюнинг точности | Корректировка промптов и правил по результатам расхождений с операторами |
| 8 недель эксплуатации | Обработка реального потока заявок в теневом режиме |
| Еженедельные сводки | Метрики точности, автоматизации и скорости — каждую неделю |
| Итоговый отчёт | Фактические KPI против согласованных, рекомендация по опции покупки |

**Что НЕ входит:** инфраструктура (предоставляет клиент), AI-API ключи (Google, Claude/Gemini — на ключах клиента, расход за период пилота ~$50–200).

**$8 000 засчитывается полностью в стоимость любой опции покупки.**

# 8. KPI пилота

Критерии успеха фиксируются письменно до старта, включая методику подсчёта и порядок разбора спорных случаев.

| Метрика | Что показывает | Целевое значение |
|---|---|---|
| Совпадение решения AI с решением оператора | Точность системы | ≥ 85% |
| Доля заявок — кандидатов на автоодобрение | Потенциал автоматизации | ≥ 50% |
| Доля документов, успешно распознанных | Применимость к реальному потоку | ≥ 90% |
| p90 времени обработки | Скорость | ≤ 15 минут |

*При частичном достижении KPI — стоимость Этапа 2 корректируется по шкале, зафиксированной в договоре пилота.*

# 10. Условия

- Цены действительны 60 дней с даты документа
